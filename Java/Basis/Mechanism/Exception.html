# 异常机制

​ 程序运行时，发生的不被期望的事件，它阻止了程序按照程序员的预期正常执行，这就是异常

# 异常的分类

​ 按照异常类将异常分为错误(Error)和异常(Exception)，而Exception中又分为 运行时异常 和 非运行时异常

  • ​ 错误：Error类以及他的子类的实例，代表了JVM本身的错误。错误不能被程序员通过代码处理，Error很少出现。
  • ​ 异常：Exception以及他的子类，代表程序运行时发送的各种不期望发生的事件。可以被Java异常处理机制处理，是异常处理的核心
    • 运行时异常(不受检异常)：RuntimeException类及其子类异常，这些异常是非检查异常，程序中可以选择捕获处理，也可以不处理。这些异常一般是由程序逻辑错误引起的，程序应该从逻辑角度尽可能避免这类异常的发生。当程序中可能出现这类异常，即使没有用try-catch语句捕获它，也没有用throws子句声明抛出它，也会编译通过
    • 编译异常(受检异常)：除 RuntimeException及其子类的Exception异常，如果程序中出现此类异常，必须对该异常做处理

​ 按照异常不同的处理方式分为 非检查异常 和 检查异常

  • 非检查异常： Error 和 RuntimeException(运行时异常) 以及它们的子类。javac在编译时，不会提示和发现这样的异常，不要求在程序处理这些异常。所以如果愿意，我们可以编写代码处理（使用try...catch...finally）这样的异常，也可以不处理。对于这些异常，我们应该修正代码，而不是去通过异常处理器处理 。这样的异常发生的原因多半是代码写的有问题。【如除0错误ArithmeticException，错误的强制类型转换错误ClassCastException，数组索引越界ArrayIndexOutOfBoundsException，使用了空对象NullPointerException等等】
  • 检查异常： 除了Error 和 RuntimeException的其它异常。javac强制要求为这样的异常做预备处理工作（使用try...catch...finally或者throws）。在方法中要么用try-catch语句捕获它并处理，要么用throws子句声明抛出它，否则编译不会通过。这样的异常一般是由程序的运行环境导致的。因为程序可能被运行在各种未知的环境下，而程序员无法干预用户如何使用他编写的程序，于是程序员就应该为这样的异常时刻准备着。【如SQLException , IOException,ClassNotFoundException 等】

# 自定义异常

​ 一般继承 Exception 或者 RuntimeException 。

public class MyException extends Exception {
    public MyException() {}
    public MyException(String msg) {
        super(msg);
    }
}

	public static void main(String[] args) {
        try {
            int divide = divide(10, 0);
            System.out.println(divide);
        } catch (MyException e) {
            System.out.println(e.getMessage());
        } finally {
            System.out.println("finally...");
        }
    }

    private static int divide(int divisor, int dividend) throws MyException {
        if (dividend == 0) {
            throw new  MyException("被除数不能为0");
        }
        return divisor / dividend;
    }

# 异常处理

# throw

​ 在程序执行过程中产生异常

使用

	if (dividend == 0) {
        throw new  MyException("被除数不能为0");
    }

​ 当方法中出现 throw 语句时，必须对异常进行处理，处理方式有：1) throws【向上抛异常】 2) try...catch【捕获处理异常】

# throws

​ 当前方法不处理异常，将该异常抛给调用 当前方法 的方法进行处理

使用

private static int divide(int divisor, int dividend) throws MyException{}

# try...catch...finally

​ try 代码块中为可能出现异常的代码，catch代码块中为当出现(捕获)到异常后 执行的代码， finally代码块中为无论是否捕获到异常都会执行的代码块【一般用于释放资源等】

使用

	try {
        int divide = divide(10, 0);
        System.out.println(divide);
    } catch (MyException e) {
        System.out.println(e.getMessage());
    } finally {
        System.out.println("finally...");
    }

​ 如果 try代码块 中捕获到异常，则将会执行对应catch中的代码，try中之后的代码不会继续执行。

​ catch (Exception e) 可以有多个，每个Exception都不相同，但可能是包含关系。如果父类Exception先被catch，那么子类Exception将永远不会被catch到。故：子类需要先被catch

​ 即使try catch 中存在return，finally代码块也会执行

return

1) return 返回值在 try 语句块中，需要没有异常时才会执行，在返回值之前会执行finally语句块，但返回值为finally语句块执行前的最终值
1) return 返回值在catch语句块中，需要捕获到对应异常才会执行，在返回值之前会执行finally语句块，但返回值为finally语句块执行前的最终值

# 常见陷阱

​ finally语句块中出现返回值，将忽视在try catch语句块中出现的throw子句

	private static void testBug(){
        try {
            int i = 10 / 0;
            throw new MyException("");
        } catch (ArithmeticException e) {
            System.out.println(e.getMessage());
            throw new MyException();
        } finally {
            return ;
        }
    }

​

​ finally语句块中出现返回值，将覆盖在try catch语句块中的return返回值，最终返回finally语句块结束后的值

	private static int testBug(){
        int i = 0;
        try {
            i = 10;
            return i;
        } catch (ArithmeticException e) {
            System.out.println(e.getMessage());
            i = 20;
            return i;
        } finally {
            i = 30;
            return i;
        }
    }

​

​ finally语句块中出现throw异常，并未使用try...catch处理【此时可以不显示抛出throws异常】，则将忽略try catch语句块中的throw return等行为

 	private static int testBug(){
        int i = 0;
        try {
            i = 10 ;
            return i;
        } catch (ArithmeticException e) {
            System.out.println(e.getMessage());
            throw new MyException();
        } finally {
            i = 30;
            throw new RuntimeException();
        }
    }

​ 此时，对于RuntimeException异常，可以不try...catch，也不throws。默认为向上抛出throws

​ 该方法不会有返回值，总是抛出RuntimeException异常
