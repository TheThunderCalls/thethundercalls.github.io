# Optional

# 概述

在编写代码的时候出现最多的就是空指针异常，所以在很多情况下我们需要做各种非空的判断，尤其是对象的属性还是一个对象的情况下，过多的判断语句会让代码显得臃肿不堪

所以在 jdk8 中引入了 Optional，使用Optional可以写出更优雅的代码来避免空指针异常，并且很多函数式编程相关的API中也用到了Optional

Optional 就好像是包装类，可以把我们的具体数据封装 Optional 对象内部。然后我们去使用 Optional 中封装好的方法操作封装进去的数据，就可以优雅的避免空指针异常

# 使用

# 创建对象 - ofNullable

方法1：Optional.ofNullable(Object value)

private static void createOption(){
    List<Author> authors = StreamTest.getAuthors();
    Optional<List<Author>> optionalAuthors = Optional.ofNullable(authors);
}

方法2：Optional.of(Object value)

private static void createOption(){
    List<Author> authors = StreamTest.getAuthors();
    Optional<List<Author>> optionalAuthors1 = Optional.of(authors);
}

只有当确定一个对象不为空的时候才可以使用该静态方法来创建Optional对象

如果传进去的数据为null，则将抛出空指针异常java.lang.NullPointerException

方法3：Optional.empty()

如果需要创建一个 null 的 Optional 对象，则可以使用该静态方法创建

# 安全消费值 - ifPresent

我们获取到一个 Optional 对象后肯定需要对其中的数据进行使用。这时可以使用 ifPresent() 来消费其中的数据

这个方法会判断其内封装的数据是否为空，不为空时才会执行具体的消费代码。

private static void testIfPresent() {
    Optional<Author> authorsOptional = createAuthorsOptional();
    authorsOptional.ifPresent(new Consumer<Author>() {
        @Override
        public void accept(Author author) {
            System.out.println(author.getName());
        }
    });
}

# 安全获取值 - orElse

如果我们期望安全的获取值，不推荐使用 get()，而是使用 Optional 提供的以下方法

  • orElseGet

获取数据并且设置数据为空时的默认值。如果数据不为空就能获取到该数据。如果为空则根据传入的参数来创建对象作为默认值返回

private static void testOrElseGet() {
    Optional<Author> authorsOptional = createAuthorsOptional();
    Author author = authorsOptional.orElseGet(new Supplier<Author>() {
        @Override
        public Author get() {
            // 如果 Optional 中的数据为null时，将返回的对象
            return new Author();
        }
    });

    System.out.println(author);
}
  • orElseThrow

获取数据，如果数据不为空就能获取到该数据，如果为空则根据传入的参数来创建异常抛出

private static void testOrElseThrow() {
    Optional<Author> authorsOptional = createAuthorsOptional();
    try {
        Author author = authorsOptional.orElseThrow(new Supplier<Throwable>() {
            @Override
            public Throwable get() {
                // 如果 Optional 中保存的数据为 null 时，将抛出的异常
                return new RuntimeException();
            }
        });
        System.out.println(author);
    } catch (Throwable throwable) {
        throwable.printStackTrace();
    }
}

# 过滤 - filter

可以使用 filter() 对数据进行过滤，如果原本是有数据的，但不符合判断，也会变成一个无数据的 Optional 对象

private static void testFilter(){
    Optional<Author> authorsOptional = createAuthorsOptional();
    Author author = authorsOptional.filter(new Predicate<Author>() {
        @Override
        public boolean test(Author author) {
            return author.getAge() < 18;
        }
    }).orElseGet(new Supplier<Author>() {
        @Override
        public Author get() {
            return new Author();
        }
    });
    System.out.println(author);
}

# 判断 - isPresent

可以使用 isPresent() 进行是否存在数据的判断。如果为空，返回值为false；如果不为空，返回值为true。

一般不使用该方法

private static void teseIsPresent() {
    Optional<Author> authorsOptional = createAuthorsOptional();
    if (authorsOptional.isPresent()) {
        System.out.println(authorsOptional.get());
    }
}

# 数据转换 - map

可以使用 map() 对数据进行转换，转换得到的数据也还是被 Optional 包装好的

例：

​ 获取作家的书籍集合

private static void testMap() {
    Optional<Author> authorsOptional = createAuthorsOptional();
    Optional<List<Book>> books = authorsOptional.map(new Function<Author, List<Book>>() {
        @Override
        public List<Book> apply(Author author) {
            return author.getBooks();
        }
    });
    books.ifPresent(new Consumer<List<Book>>() {
        @Override
        public void accept(List<Book> books) {
            books.forEach(book -> System.out.println(book));
        }
    });
}
